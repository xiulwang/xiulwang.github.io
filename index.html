word2vec中的两个模型

如图所示，CBOW是已知当前词$w_t$的上下文$context(w_t)$，预测当前词$w_t$；SKip-gram恰好相反。

CBOW（Continuous Bag-of-Words Model）

基本结构

  • Input Layer
    图中的 $context(w)$ 包含 $2c$ 个词向量：$v(context(w)_i)\in \mathbb{R}^m$
  • Projection Layer
    将输入层的 $2c$ 个词向量做累加，即 $X_w = \sum_{i=1}^{2c}v(context(w)_i) \in \mathbb{R}^m$
  • Output Layer
    Huffman树，叶子节点对应于词典 $D$ 中的每一个词，$N=|D|$，非叶子节点$N-1$个。

条件概率$p(w|context(w))$

word2vec中，编码为0被定义为正类，1为负类(左节点为正，右节点为负)
$$label(p_i^w)=1-d_i^w, i=2,3,…,l_w$$
对于每一个节点，被分为正类的概率为：
$$\sigma(X_w^ \mathrm{T}\theta)=\frac{1}{1+e^{-X_w^ \mathrm{T}\theta}}$$
负类概率：
$$1-\sigma(X_w^ \mathrm{T}\theta)$$
对于词典 $D$ 中的任意一个词，Huffman树中都存在唯一一条从根节点到词 $w$ 的一条路径 $p^w$, 一共有 $l^w-1$ 个分支，每一次分支即为一次二分类。条件概率 $p(w|context(w))$ 被定义为
$$p(w|context(w))=\prod_{j=2}^{l^w}p(d_j^w|X_w,\theta_{j-1}^w)$$
其中
$$p(d_j^w|X_w,\theta_{j-1}^w)=
\begin{cases}
\sigma(X_w^ \mathrm{ T }\theta_{j-1}^w)& d_j^w=0\\
1-\sigma(X_w^ \mathrm{ T }\theta_{j-1}^w)& d_j^w=1
\end{cases}$$
可将其合并：
$$p(d_j^w|X_w,\theta_{j-1}^w)=[\sigma(X_w^ \mathrm{ T }\theta_{j-1}^w)]^{1-d_j^w}\cdot [1-\sigma(X_w^ \mathrm{ T }\theta_{j-1}^w)]^{d_j^w}$$
$\sum_{w\in C}p(w|context(w))=1$ ，可以参考Hufffman树的结构，因为叶子节点都对应于每一个词，每两个词的概率相加等于它们的父节点，最后所有词相加即为根节点

随机梯度上升

$$L = \sum_{w\in C}log\prod\ _{j=2}^{l^w}{[\sigma(X_w^ \mathrm{ T }\theta_{j-1}^w)]^{1-d_j^w}\cdot [1-\sigma(X_w^ \mathrm{ T }\theta_{j-1}^w)]^{d_j^w}}$$
$$L = \sum_{w\in C}\sum_{j=2}^{l^w}({(1-d_j^w)\cdot log[\sigma(X_w^ \mathrm{ T }\theta_{j-1}^w)]+d_j^w\cdot log [1-\sigma(X_w^ \mathrm{ T }\theta_{j-1}^w)]})$$

其中$w$代表字典中的一个词，$j$代表根节点到该词路径上的一个节点。为了使得概率最大化,对于一个随机样本 $(context(w),w)$ 求参数 $x_w,\theta_{j-2}^w$的偏导，这里的 $X_w$ 是 $context(w)$ 的词向量的累加，$\theta_{j-2}^w$ 对应于节点向量:

word2vec-基础语言模型

发表于 2018-04-24 | 分类于 NLP

1. n-gram模型

核心假设：一个词出现的概率只与它前面的n个词相关，即：
$$P(w_k|w_1,…,w_{k-1}) \approx P(w_k|w_{k-n+1},…,w_{k-1})$$
根据大数定律，当语料库足够大时，概率可以由以下公式得出：
$$P(w_k|w_1,…,w_{k-1}) = \frac{count(W_{k-n+1},…,W_{k})}{count(W_{k-n+1},…,W_{k-1})}$$
这样可以使得参数更少，统计更加容易。

  • unigram：n=1
  • bigram: n=2
  • trigram: n=3

平滑：以下情况需要平滑
$count(W_{k-n+1},…,W_{k})=0$
or
$count(W_{k-n+1},…,W_{k})=count(W_{k-n+1},…,W_{k-1})$

总结：利用语料中各个词汇出现的次数，计算相应的概率值，用于对一个新词的预测。

2. 机器学习–神经网络

2.1. 核心目标：最小化损失函数

$(context(w), w)$对应于一个训练样本，似然函数最大化：
$$\prod_{w \in C}P(w|context(w))$$
其中，$C$为语料库，context(w)为$w$的上下文信息。在n-gram中，对应于前n个词的个数。
$$L = \sum_{w \in C}log(w|context(w))$$

2.2. 词向量

目标：将语言向量化，以便机器处理

2.2.1. one-hot representation

最简单的一种方式，假设字典 $D$ 的大小为 $N$, 对于字典(语料 $C$ 中不重复的词)中的每个元素，都对应于一个 $N$维的向量，该向量中只有一个位置为1，其余都为0，1的位置即是该元素的索引。

缺点：稀疏存储，维度高；会造成任意两个词之间的孤立性

2.2.2. distributed representation

把每一个词映射成一个固定长度的短向量，在这个词向量的空间中，根据词之间的距离来判断它们之间的相似性。这里的词可以替换为句子或者文档。
常见的方法：LSA, LDA， 神经网络……

2.3. 神经网络模型

  • Input Layer:
    $$X_w = \bigl(\begin{smallmatrix}
    v(context(w)_1)\
    .\
    .\
    .\
    v(context(w)_{n-1})
    \end{smallmatrix}\bigr)$$
    其中$w(x)$ 代表词$x$ 的词向量，$X_w\in \mathbb{R}^{(n-1)*m}$, $m$ 为词向量的维度。
  • Hidden Layer:
    $$z_w = tanh(WX_w+p)$$
    其中$W\in \mathbb{R}^{n_h\times (n-1)m}, p\in \mathbb{R}^{n_h}$

  • Output Layer：
    $$y_w = Uz_w + q$$
    其中$U\in \mathbb{R}^{N\times n_h}, q\in \mathbb{R}^N$, $N$代表了字典$D$的大小。

  • 字典 $D$ 中每一个词的概率为：
    $$p(w|context(w)) = \frac{e^{y_{w,i_w}}}{\sum_{i=1}^Ne^{y_{w,i}}}$$
    其中 $i_w$ 为词 $w$ 在词典$D$中的索引。

优势：1. 相似的词或者句子的概率很相近；2. 平滑处理

Hadoop初试

发表于 2018-04-20 | 分类于 大数据

1. HDFS(Hadoop Distributed File System)

HDFS架构

  • 数据块：读/写的最小单位，HDFS的块更大，128MB, 是为了最小化寻址开销。
  • Namenode: 一个管理节点，管理文件系统树及整棵树内的文件和目录。
  • Datanode: 多个工作节点，记录每个文件中各个块所在的数据节点信息，Datanode从磁盘中读取块。

HDFS操作

  • hadoop fs -ls
  • hadoop fs -put localfile hdfsfile 上传本机文件到hdfs中的目录下
  • hadoop fs -get hdfsfile localfile 下载hdfs中的文件至本机
  • hadoop fs -test -e path 检查文件是否存在，如果存在，返回0
  • 参考 https://www.yiibai.com/hadoop/hadoop_command_reference.html#article-start

2. Map-Reduce

三个阶段：Map,Shuffle,Reduce.

Map阶段

输入数据存储在HDFS中，处理后形成若干小块。

Shuffle阶段

Map阶段后，对key值进行了排序，再交给reduce，这个过程系统自动实现。

Reduce阶段

聚拢数据再处理，将输出结果返回到Hadoop服务器上。

3. Streaming

作用

任何语言编写的map, reduce程序能够在hadoop集群上运行，map/reduce程序只要遵循从标准输入stdin读，写出到标准输出stdout即可。

命令

hadoop  jar /hadoop-mapreduce/hadoop-streaming.jar \\
-D mapred.job.name="xxx" \\
-D mapreduce.job.queuename= xxx\\
-D mapred.map.tasks= x\\
-D mapred.reduce.tasks= x\\
-cmdenv CLASSPATH=\\
-input  xxx\\
-output  xxx\\
-mapper "python mapper.py" \\
-reducer "python reducer.py" \\
-file mapper.py \\
-file reducer.py

4. Python处理标准输入/输出相关

  • sys.argv: 用户输入的参数列表，程序外部输入
  • os.system(‘command’): 会执行括号中的命令；如果成功，返回0，否则返回1；执行多条语句时，用;分隔
  • sys.exit(1): 有错误退出
  • sys.exit(0): 无错误退出
  • str.join(sequence): str是连接的符号，sequence是连接的字符串序列
  • 调试时使用管道，模拟streaming, 在本地完成map/reduce程序的调试

模型评估

发表于 2018-03-02 | 分类于 机器学习

1. 训练集划分

留出法

  • 把数据集D划分为两个互斥的子集
  • train/test数据集应该尽可能保持数据分布的一致性，避免因为划分而引入额外的偏差（使用分层采样）
  • 单次使用因为随机划分会导致估计结果不稳定，应该多次随机划分，重复进行实验评估后取平均值。

交叉验证法

  • 将数据集D划分成k个互斥的子集，每次使用k-1个集合数据作为Train，剩下的子集作为test，从而进行k次训练和测试，返回均值。
  • 评估结果依赖于k的选择。
  • 留一法是交叉验证的特殊情况；优点：使用了尽可能多的数据进行训练；缺点：计算开销

    自助法（Bootstrapping）

  • 每次从数据集D中选择一个样本拷贝到D’,直至|D|=|D’|，D’用于训练，D-D’用于测试。
  • $\lim_{m\rightarrow+\infty}(1-\frac{1}{m})^m = \frac{1}{e}$, 样本在m次采样中始终不被采到的概率，包外估计
    优点：适用于数据集过小时；产生了多个不同的训练集，适用于集成学习
    缺点：改变了数据的原始分布，会带来估计得偏差。

2. 性能度量

真实/预测 正例 反例
正例 TP FN
反例 FP TN

PR曲线

  • Precision: $\frac{TP}{TP+FP}$
  • Recall: $\frac{TP}{TP+FN}$
    查准率和查全率是一对矛盾的度量，设定一个threshold，就可以得到相应的P和R，得到PR曲线
  • BEP：平衡点 P=R时的取值
  • F1度量：$$\frac{1}{F_1}=\frac{1}{2}*(\frac{1}{P}+\frac{1}{R})$$
  • $$\frac{1}{F_\beta}=\frac{1}{1+\beta^2}*(\frac{1}{P}+\frac{\beta^2}{R})$$
  • $\beta>1$时，查全率影响更大

ROC和AUC

  • ROC的纵轴：TPR(True Positive rate)
  • ROC的横轴：FPR(False Positive rate)
  • $TPR=\frac{TP}{TP+FN}$(Recall)
  • $FPR=\frac{FP}{FP+TN}$
  • AUC为ROC曲线下的面积

代价敏感

  • 不同类型错误所造成的损失不同。

3. 偏差与方差

  • 泛化误差=偏差+方差+噪声
  • 噪声是期望误差的下限
